🔍 Контекст

Яндекс Карты — один из крупнейших картографических сервисов России с многомиллионной аудиторией. Справка мобильного приложения — основная точка самообслуживания: сюда попадают пользователи, которые не могут разобраться с функциями продукта или столкнулись с проблемой.

Я работала в команде технических писателей Яндекса в роли редактора и руководителя редакции. В числе моих задач была поддержка и улучшение документации для пользовательских продуктов компании. Одной из ключевых задач была переработка структуры справки мобильных Яндекс Карт на основе данных.

❗ Проблема

Перед началом работы изучила метрику и выявила системные проблемы:

  • Пользователи не находили нужные статьи — основной источник трафика в справку это поисковые системы, но заголовки статей не совпадали с реальными поисковыми запросами пользователей.
  • Структура не соответствовала пути пользователя — например, 4 отдельные статьи про офлайн-карты вместо одной понятной; статьи про «обычную линейку» и «масштабную линейку» были разделены, пользователи постоянно путали их и переходили между ними.
  • Часть функций не была видна в поиске вообще — страница про Алису была спрятана внутри статьи про настройки приложения, поисковых переходов на неё не было совсем.
  • Несоответствие названий и запросов — статья называлась «Карты без интернета», а пользователи искали «офлайн карты»; статья «Версия приложения» описывала обновление, а не версию.
  • Команда саппорта получала типовые вопросы, ответы на которые отсутствовали в справке: как удалить приложение, как поделиться координатами, как решить технические проблемы.

🎯 Задача

Переработать структуру и содержание справки мобильных Яндекс Карт так, чтобы:

  1. Пользователи находили ответы через поисковые системы по своим реальным запросам.
  2. Снизилась нагрузка на команду саппорта по типовым вопросам.
  3. Справка отражала актуальное состояние продукта.

⚙️ Что я сделала

1. Анализ аудитории и метрики справки

Изучила данные Яндекс Метрики и составила портрет аудитории:

  • Основная аудитория — в среднем 25–44 лет. Вывод: статьи должны быть написаны простым языком с подробными пошаговыми инструкциями.
  • Справку читают преимущественно с мобильных устройств — значит, статьи не должны быть перегружены, настройки лучше разбивать по отдельным сценариям.
  • Большинство пользователей из России, но также есть и из других стран — нужно указывать ограничения в работе некоторых функций.
  • Основной трафик приходит из поиска — поэтому поисковые запросы стали главным ориентиром при работе с заголовками.

Выявила топ популярных страниц и топ непосещаемых: непосещаемые страницы дополнительно проверили на актуальность, реструктуризировали в справке.

2. Анализ поисковых запросов (Метрика + Wordstat)

Сопоставила реальные поисковые запросы с заголовками статей справки. Нашла критические расхождения, например:

  • Пользователи искали «офлайн карты», а статья называлась «Карты без интернета».
  • Запросы «скачать яндекс карты» вели не на страницу установки приложения, а на скачивание офлайн-карт.
  • По запросам «обновить яндекс карты», «удалить приложение» справка не появлялась в результатах.
  • Статья про Алису не индексировалась — поисковых переходов на неё не было вовсе.

3. Анализ базы знаний саппорта

Изучила обращения в поддержку, нашла типовые вопросы, которые не были покрыты в справке. На основе этого:

  • Добавила страницу «Решение проблем» с самыми частыми техническими сценариями — чтобы пользователи сначала пробовали решить вопрос самостоятельно, а потом шли в поддержку.
  • В статьи про транспорт, парковки и офлайн-карты добавила блоки с ответами на частые вопросы.

4. Переработка структуры и содержания

На основе данных внесла конкретные изменения, например:

Установка приложения

  • Переименовала подзаголовок в «Скачать приложение» — по запросу «скачать яндекс карты» пользователи стали попадать на нужную страницу.
  • Выделила отдельным подзаголовком инструкцию по обновлению.
  • Добавила инструкции «Как посмотреть версию приложения» и «Как удалить приложение».

Офлайн-карты

  • Объединила 4 отдельные статьи в одну.
  • Переименовала «Карты без интернета» → «Карты офлайн» в соответствии с реальными поисковыми запросами.

Линейка и измерение расстояния

  • Объединила описание обычной и масштабной линейки в одну статью — пользователи переходили между ними.

Алиса

  • Вынесла описание в отдельную статью вместо раздела внутри настроек.
  • Добавила инструкцию про звонки с Алисой.

Решение проблем

  • Создала новую страницу с типовыми техническими сценариями на основе обращений в саппорт.

5. Обеспечение актуальности

  • Согласовывала изменения с командой саппорта и заказчиками.
  • Поддерживала локализованные версии документов в актуальном состоянии.
  • Настройка редиректов привела к снижению показателей отказов по справке.

📊 Результат

Общий трафик (период 2023–2024 vs 2022–2023):

  • Переходы из поисковых систем выросли на 3,6%
  • Переходы из Мобильного Яндекса выросли на 6%

По конкретным страницам:

  • Установка — пользователи по запросам «скачать яндекс карты» и «скачать приложение карты» стали попадать на нужную страницу; посещаемость страницы выросла.
  • Офлайн-карты — переходы по запросу «скачать яндекс карты» на эту страницу прекратились; зато появился значительный поток по запросам «офлайн карты».
  • Линейка — пользователи стали приходить по обоим видам запросов («линейка» и «измерить расстояние») на одну страницу.
  • Алиса — новая страница проиндексировалась, появился стабильный поток пользователей из поиска по запросам про работу Алисы в Картах.
  • Решение проблем — новая страница стабильно получает трафик, запросы пользователей совпадают с описанием статьи.

💡 Вывод

Пользователи формулируют вопросы иначе, чем команда описывает функции продукта. Разрыв между языком продукта и языком пользователя — это и есть основная зона работы UX-редактора.

Хорошая справка — это не описание функционала. Это ответы на вопросы, которые пользователь уже несёт в голове, когда открывает поиск. Данные Метрики и Wordstat позволяют понять этот язык и перевести на него контент.
